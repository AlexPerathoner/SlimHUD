Getting started

SlimHUD is a tiny application that lives in your Mac's Menu Bar. After the installation you probably won't touch it anymore and it will just become part of your workflow.

SlimHUD was born after the release of iOS 13 in which the volume hud finally changed. On MacOS however, the hud is still the same as ages ago so I decided to create an alternative.

SlimHUD aims to bring on your Mac the same GUI experience we already have on our iPhones and iPads.

Launching SlimHud is easy: copy the app into your Application's folder and then open it.
However, as you probably also want it to replace MacOS' HUDs we'll have to follow a few steps more...
Please note that the following instructions will temporarily deactivate Sysem Integrity Protection. By proceeding you acknowledge that you are aware of which risks this leads to.
But do not be scared, just follow everything and you will be fine (source: here and here):

  1. Enter Recovery Mode by restarting your mac and holding ⌘+R while it is booting up.
  2. Launch the terminal by clicking on Utilities > Terminal.
  3. Disable SIP with csrutil disable.
  4. Restart your mac and login into an admin account.
  5. Run sudo -uw /; killall Finder in your terminal.
  6. Now run sudo nano /System/Library/LaunchAgents/com.apple.OSDUIHelper.plist.
  7. Remove all of the text between and . Save the file by hitting control + X, answering yes and hitting return.
  8. Re-enter Recovery mode.
  9. Enable SIP with csrutil enable.

You have now successfully replaced those ugly and old overlays! Congrats!

For future updates it is not necessary to go through all of these steps. Just copy the latest release into your application Folder or check for new updates directly from inside the app.

FAQ

What is step 5 for?

sudo allows a permitted user to execute a command as the superuser or another user. Mount is used to mount disks. The -u flag indicates that the status of an already mounted file system should be changed. -w means to mount the file system read-write. The file system is "/" aka root. The killall utility kills processes selected by name. Since it's Finder, it will just restart and reflect the change you've just made. This is not permanent, there are more steps involved for that.
Source: reddit

This is needed as in the last versions of MacOS the system files (like /System/Library/LaunchAgents/com.apple.OSDUIHelper.plist that you'll modify in step 6) are stored in a different disk partition and being an administrator and having System Integrity Protection disabled still isn't enough to modify them.

PS: note that the right command is sudo mount -uw /, not sudo mount -uw

Will I harm my Mac installing SlimHUD?

No, you will not.
I reccomend to only do what you need to do while having SIP disabled and as soon as you finish the installation re-enable SIP. Once enabled everything will be back to normal.

Settings

The settings window is made up of 3 sections:
  • General
  • Position
  • Style
Remember that a preview of the bars is always visible. In addition, the volume bar is kept visible on the screen.
This should give you a exhaustive idea of what the look will be like.
  • Bars: choose individually which HUDs should appear;
  • Margin-value: to improve the energy footprint SlimHUD only shows a HUD when the value has changed by more than this given value;
  • Check for Updates: checks if a new update has been released;
  • Continuously check: SlimHUD usually observes if a function key has been pressed and then displays a HUD. However, there are situations in which the value changes without a key being pressed. This usually happens when you use a slider on the TouchBar. Therefore you can enable this setting. Remember that this will lead to a small, but present, increase in the energy footprint;
  • Launch at login: will automatically launch SlimHUD when you boot up your Mac;
  • Position: choose the edge the HUDs should appear on;
  • Size: set the main size of the bar;
  • Thickness: set the thickness of the bar;
Note that the preview on the left will not display changes concerning this settings. The actual volume bar is kept visible and you will be able to see the changes directly there.
  • Icons: icons can be shown or hidden. There is a icon for the volume, one for the brightness and one for the keyboard backlight;
  • Animations: toggles the animation displayed when the hud appears on the screen, and when the value changes while the bar is visible. This setting doesn't affect the energy footprint so feel free to choose what you like the most;
  • Shadows: toggles a shadow effect behind the bar. This can help to improve the visibility of the bar;
  • Colors: customise the colors of each bar and icon. The background color is the same for every bar.
    The volume has two settable colors as it can be muted;
