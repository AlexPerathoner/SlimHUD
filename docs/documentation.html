Getting started

SlimHUD is a tiny application that lives in your Mac's Menu Bar. After the installation you probably won't touch it anymore and it will just become part of your workflow.

SlimHUD was born after the release of iOS 13 in which the volume hud finally changed. On MacOS however, the hud is still the same as ages ago so I decided to create an alternative.

SlimHUD aims to bring on your Mac the same GUI experience we already have on our iPhones and iPads.

Installation

Move the application file to the /Application folder

Uninstallation

Move the application file to the trash bin. Log in again to restore MacOS’s Huds.

Some questions?

Open an issue on GitHub!

Settings

The settings window is made up of 3 sections:
  • General
  • Position
  • Style
Remember that a preview of the bars is always visible. In addition, the volume bar is kept visible on the screen.
This should give you a exhaustive idea of what the look will be like.
  • Bars: choose individually which HUDs should appear;
  • Margin-value: to improve the energy footprint SlimHUD only shows a HUD when the value has changed by more than this given value;
  • Check for Updates: checks if a new update has been released;
  • Continuously check: SlimHUD usually observes if a function key has been pressed and then displays a HUD. However, there are situations in which the value changes without a key being pressed. This usually happens when you use a slider on the TouchBar. Therefore you can enable this setting. Remember that this will lead to a small, but present, increase in the energy footprint;
  • Launch at login: will automatically launch SlimHUD when you boot up your Mac;
  • Position: choose the edge the HUDs should appear on;
  • Size: set the main size of the bar;
  • Thickness: set the thickness of the bar;
Note that the preview on the left will not display changes concerning this settings. The actual volume bar is kept visible and you will be able to see the changes directly there.
  • Icons: icons can be shown or hidden. There is a icon for the volume, one for the brightness and one for the keyboard backlight;
  • Animations: toggles the animation displayed when the hud appears on the screen, and when the value changes while the bar is visible. This setting doesn't affect the energy footprint so feel free to choose what you like the most;
  • Shadows: toggles a shadow effect behind the bar. This can help to improve the visibility of the bar;
  • Colors: customise the colors of each bar and icon. The background color is the same for every bar.
    The volume has two settable colors as it can be muted;
